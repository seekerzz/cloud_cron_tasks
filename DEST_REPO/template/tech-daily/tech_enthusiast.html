🔥 今日头条

AI也能做梦了！Claude Code推出AutoDream功能，让AI像人类一样"睡眠整理记忆"
Anthropic为Claude Code推出了一个名为AutoDream的新功能，这个名字来源于神经科学中的"睡眠记忆巩固"现象。就像人类在睡眠时大脑会整理白天的经历、强化重要连接、修剪无用信息一样，AutoDream会在你不使用Claude的时候自动运行，执行三项任务：删除重复和过期的记忆、将散落的信息合并整理、用项目当前状态刷新旧记忆。这解决了长期使用AI助手的一个痛点——记忆文件越来越臃肿，有用信息被噪音淹没。与Cursor的Continual Learning不同，AutoDream选择在会话之间运行，不占用用户工作时的资源，真正实现了"该干活时干活，该做梦时做梦"。
AI创新 来源：Orange AI(@oran_ge)
飞书开源CLI工具，让任何AI Agent都能操控飞书全套功能
飞书正式开源了其lark-cli命令行工具，这是一个专门为人类和AI Agent设计的工具，覆盖了消息、文档、多维表格、电子表格、日历、邮箱、任务、会议等11个核心业务域，提供200+命令及19个AI Agent Skills。安装后，你可以直接跟Claude Code或Codex说"帮我查下周日程"、"把这份Markdown转成飞书文档"、"分析我过去一个月的文档并创建仪表盘"，AI会自动调用飞书API完成操作。这标志着企业协作软件正式进入AI Agent原生时代，飞书也因此成为国内对AI Agent最友好的产品之一。
产品更新 来源：赛博禅心
17岁高中生自学编程，用AI解决反盗猎难题，准确率超越Google
圣地亚哥的17岁高中生Naveen Dhar自学编程，开发了一套AI系统来检测丛林中的偷猎枪声。此前Google在喀麦隆的尝试中，1700多个声音警报中只有3个是真的枪声，误报率高达99.8%，导致护林员对系统失去信任。Dhar从头开始构建自己的AI，用伯利兹的声音数据训练，在非洲和越南测试时几乎零误报，而且无需针对新环境重新训练。他已经在全球最大的AI会议之一上展示了这个项目——而那时他还没高中毕业。这个故事告诉我们：技术能力正在民主化，一个 motivated 的个体可以解决连科技巨头都搞不定的问题。
科技向善 来源：The Rundown AI(@TheRundownAI)
OpenAI Codex技术负责人：AI已能写80%代码，但"问对问题"比"会写代码"更重要
OpenAI Codex技术负责人Michael Bolin在最新播客中分享了一个关键洞察：在AI编程时代，80%-90%的代码已经可以由模型生成，但真正拉开差距的从来不是写代码的速度，而是你选择解决什么问题，以及如何定义"更好的系统"。他观察到，很多工程师正在用AI做"以前做的事，只是更快"，而真正的机会在于用AI做"以前不敢想的事"。随着Agent能力增强，提出正确问题的能力、系统架构设计能力、以及理解底层原理的能力，将比单纯的编码能力更有价值。
行业洞察 来源：AI前线
西湖大学开源AutoFigure-Edit：AI生成的论文插图终于可以编辑了
西湖大学张岳实验室推出了AutoFigure-Edit系统，解决了AI生成学术插图"生成后无法修改"的痛点。与之前生成静态PNG不同，新系统输出的是完全可编辑的SVG文件，你可以在浏览器内直接拖拽、改字、换色。系统还支持"风格迁移"——上传一张你喜欢的风格图，AI会自动学习其配色、字体、图标风格并应用到你的科学插图上。对于科研工作者来说，这意味着论文插图工作流将被彻底改变：从"反复生成-挑选-放弃修改"变成"生成-微调-完美输出"。
科研工具 来源：Datawhale

🎓 科普小课堂

📚 什么是"Vibe Coding"（氛围编程）？

概念解释： "Vibe Coding"是最近AI编程圈流行的一个新词，指的是一种全新的编程方式——工程师不再逐行编写代码，而是用自然语言描述需求，让AI Agent来完成实现。你只需要"vibe"（感受/氛围）对了，把想法告诉AI，剩下的交给它。

为什么重要：

  • 编程民主化：不懂代码的人也能创建应用
  • 效率提升：工程师从"写代码"转向"指挥AI"
  • 思维转变：重点从"怎么实现"变成"要解决什么问题"

有趣事实： Cognition的工程师表示，他们现在已经"不打代码了"——管理AI Agent一小时，抵得上自己干6到12小时。你几乎成了一支Agent军团的CTO，只需要定义问题域，它们去执行。
